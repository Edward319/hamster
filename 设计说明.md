## 二、页面与导航（已定稿）

根据你的选择，整理如下。

### 2.1 底部四个入口

| 入口 | 名称 | 内容 |
|------|------|------|
| 1 | **首页** | 第一眼看到的「今日提醒」：即将过期产品总结，用**仓鼠贺卡**的 UI 表现（小仓鼠 + 提醒卡片） |
| 2 | **库存管理** | 添加、修改、标记存货属性和状态，查看所有存货。UI：像**仓库申请卡片**一样，在**仓鼠卡通管理员**提供的可爱表格上填写/操作数据 |
| 3 | **报告** | 定期周报、月报。报告页内用**底部 Tab** 切换「周报」「月报」 |
| 4 | **设置** | 通知周期、提醒周期，是否接入 Notion，数据选线下还是线上同步，以及个性化通知相关选项 |

### 2.2 首页与报告的结构

- **首页**：只放**临期相关**——今日/即将过期产品总结 + 仓鼠贺卡式呈现。
- **报告**：周报、月报都放在这里；进入「报告」后通过**底部 Tab**（或分段）在「周报」「月报」之间切换。

### 2.3 页面列表小结

| 页面 | 从哪进 | 主要元素 |
|------|--------|----------|
| 首页 | 底部「首页」 | 仓鼠贺卡 UI、今日/即将过期产品总结 |
| 库存管理 | 底部「库存管理」 | 仓鼠管理员 + 仓库申请卡片式表格；支持添加、编辑、标记状态、查看全部存货 |
| 报告（周报） | 底部「报告」→ 周报 Tab | 周维度报告内容 |
| 报告（月报） | 底部「报告」→ 月报 Tab | 月维度报告内容 |

## 三、添加/编辑货物流程（已定稿）

根据你的选择，整理如下。

### 3.1 表单方式

- **一页填完**：物品名称、一级品类、二级品类、品牌、数量、购买日期、保质期/赏味期、单位价格、备注等，在同一页（同一张「仓库申请卡片」）内**一口气填完**再提交，不分步。

### 3.2 品类选择方式

- **支持自己输入品类，自动保存，下次可选**：一级、二级品类都支持用户输入新名称；输入后自动保存到「我的品类」库，下次在同一位置可从下拉或列表中选择已有品类，也可继续输入新的。无需预先穷举，完全按个人习惯积累。

### 3.3 添加入口与复制功能

- **添加入口**：在**库存管理页顶部**有「添加」按钮，点击后进入**全新空白表单**。
- **复制已有库存**：表单内提供「**从已有库存复制**」功能（如下拉选一条已有记录，或从库存列表某条点「再买一次」）。复制后带出该条的名称、一级/二级品类、品牌等，用户只需改数量、购买日期、保质期等，减少重复录入。适合经常买同一种产品的情况。

### 3.4 小结（开发时可对照）

| 项目 | 结论 |
|------|------|
| 添加入口 | 库存管理页顶部「添加」按钮 |
| 表单形态 | 一页、仓库申请卡片式、一口气填完 |
| 品类 | 可输入新品类，自动保存，下次可选（一级、二级均支持） |
| 复制 | 支持从已有库存复制信息，再改数量/日期等 |

## 五、小仓鼠管家出场时机与动作（已定稿）

根据你的选择，整理成「小仓鼠出场清单」，开发时可逐条实现。

### 5.1 首页（今日提醒 / 仓鼠贺卡）

- **有即将过期时**：仓鼠像**员工**一样给用户**递上列表**（提醒列表由仓鼠「递上」的姿势/构图呈现）。
- **没有即将过期时**：使用默认文案：**「今天没有快过期的东西，真棒！」**（可配合仓鼠递上一张「空列表」或轻松表情）。

### 5.2 库存管理（列表与操作反馈）

- **进入列表时**：仓鼠像**管理员**一样给用户**递上表格**（表格由仓鼠「递上」的视觉呈现）。
- **添加完成 或 标记已使用完 时**：仓鼠像管理员一样，在**表格上面有一个盖章动作**（如「已登记」「已使用」等盖章效果），作为操作成功的反馈。

### 5.3 报告页（周报 / 月报）

- **打开周报或月报时**：小仓鼠**举着报告**，展示对应周报或月报内容。

### 5.4 其他场景

- **暂不增加**；后续若有需要（如删除确认、设置保存成功等）可再补充到本清单。

### 5.5 小结表（开发对照）

| 场景 | 仓鼠角色/动作 | 备注 |
|------|----------------|------|
| 首页有临期 | 像员工，递上列表 | 贺卡式 UI |
| 首页无临期 | 默认文案：「今天没有快过期的东西，真棒！」 | 可配合递空列表或轻松表情 |
| 库存管理-进入列表 | 像管理员，递上表格 | 仓库申请卡片式表格 |
| 添加完成 / 标记已使用完 | 像管理员，在表格上盖章 | 操作成功反馈 |
| 报告（周报/月报） | 举着报告 | 展示报告内容 |
| 其他 | 暂不增加 | 后续可补充 |

## 六、手帐风 UI 方向（已定稿）

根据你的选择，整理为可落地的 UI 要点，开发时统一视觉语言。

### 6.1 配色

- **偏暖**：主色以米白、浅棕、淡黄、藕粉等暖色为主，整体给人温暖、好亲近的感觉，与「家庭存货」「手帐」调性一致。避免冷峻的灰蓝、冷白大面积使用。

### 6.2 纸质感

- **手帐感觉，不强调真实纸纹**：不追求逼真纸张纹理或厚重阴影，而是通过**圆角、柔和、留白**等营造手帐感。界面块面圆润、边角柔和，整体轻盈不沉重。

### 6.3 便签 / 胶带 / 手写元素

- **多用**：便签块、胶带装饰、手写感字体等可以**较多使用**，让界面有明显的手帐装饰感。例如：卡片像便签、分区用胶带或手绘线分隔、标题或提示用圆润/手写感字体。

### 6.4 参考

- **暂不指定**；若后续有喜欢的参考（手帐 App、插画、风格词）可再补充，微调配色或元素密度。

### 6.5 小结（开发对照）

| 项目 | 结论 |
|------|------|
| 配色 | 偏暖：米白、浅棕、淡黄、藕粉等 |
| 纸感 | 手帐感觉：圆角、柔和、留白，不强调真实纸纹 |
| 便签/胶带/手写 | 多用：便签块、胶带、手写感字体 |
| 参考 | 暂无，后续可补充 |

## 七、邮件版式与风格（已定稿）

根据你的选择，整理为邮件版式要点，与手帐风、小仓鼠统一。

### 7.1 整体风格

- **更像小报/简报**：邮件整体像「一张小报/简报」——标题 + 区块列表，信息清晰、易扫读，而不是长信式段落。
- **要信纸边框和底纹**：保留**信纸边框**和**底纹**（与手帐风、暖色一致），让邮件有「从信箱里拿出来」的仪式感，又不影响可读性。

### 7.2 小仓鼠在邮件中

- **每封都出现**：临期提醒、周报、月报**每封邮件**里小仓鼠都出现（如站在信头或举着内容），保持品牌与互动感统一。

### 7.3 三种邮件版式

- **暂时只换内容**：临期提醒 / 周报 / 月报 使用**同一套版式**，只替换标题与正文内容。后续若需要区分（如月报更「总结感」）可再迭代。

### 7.4 小结（开发对照）

| 项目 | 结论 |
|------|------|
| 整体形态 | 小报/简报：标题 + 区块列表 |
| 信纸 | 有边框、有底纹（暖色/手帐风） |
| 小仓鼠 | 每封都出现 |
| 临期/周报/月报 | 同一套版式，只换内容 |
