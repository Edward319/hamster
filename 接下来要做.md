# 接下来要做

> 当前版本已具备：首页今日提醒、库存管理、货单总结（可配置周期）、设置（通知/数据保存/自动清理）、Notion 同步、**发送今日报告到邮箱**（含邮件内「过去 N 周」统计、按通知频率进入首页时的发送提醒）、邮件配置说明（Resend/Gmail SMTP）。  
> 下面分「上线与分享」和「功能待办」两块。

---

## 一、上线与分享（把当前版本给其他人用）

按《下一步操作清单》顺序做即可：

| 顺序 | 要做的事 | 说明 |
|------|----------|------|
| **1** | **推送到 GitHub** | 在项目目录执行 `git push origin main`（需本机登录 GitHub 或配置 SSH）。代码已在本地提交好，只差推送。 |
| **2** | **Vercel 部署** | 若还没部署：Vercel 用 GitHub 导入 `Edward319/hamster`，Deploy。得到网址如 `https://xxx.vercel.app`。 |
| **3** | **配置发邮件（Resend）** | 在 Resend 创建 API Key → 在 Vercel 项目里加环境变量 `RESEND_API_KEY` → 重新部署一次。 |
| **4** | **自测** | 打开部署好的网址 → 设置里填自己的邮箱 → 首页点「发送今日报告到邮箱」→ 检查是否收到邮件。 |
| **5** | **分享** | 把 Vercel 网址发给别人，他们打开即可用；各自在设置里填自己的邮箱，数据互不影响。 |

做完 1～5，别人就可以用当前版本，并收到自己的报告邮件。

---

## 二、功能待办（后续迭代）

| 类型 | 项 | 说明 |
|------|----|------|
| **邮件** | **服务端定时自动发** | 已实现：首页手动发送 + 进入首页时按「通知频率」提醒发送。若要做「到点自动发、用户不打开应用也收信」，需 Vercel Cron + 后端存用户邮箱与数据源（或 Notion 令牌），成本与复杂度较高，可后续再考虑。 |
| **邮件** | **条件触发提醒** | 需求里提到的「有货品进入 3 天内过期时自动发一封临期提醒」等，可与定时邮件一起设计。 |
| **管家** | **AI 管家** | 管家页目前是占位，后续可接入大模型（DeepSeek/千问/豆包等），用户连自己的 API，读取库存做对话与建议。 |
| **体验** | **PWA / 添加到主屏幕** | 可选：让用户把网址「添加到主屏幕」，更像 App。 |
| **其他** | 见《需求说明》第一版验收标准 | 大部分已满足；未完全满足的可按该文档逐项补。 |

以上不影响「把当前版本发给别人用」；先完成「一、上线与分享」，再按需做「二、功能待办」即可。
